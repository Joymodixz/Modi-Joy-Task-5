# Task 5 - Capture and Analyze Network Traffic Using Wireshark

## Objective
Capture live network traffic using Wireshark and analyze different network protocols.

## Tools Used
- Wireshark
- Windows 11
- Internet Connection

## Steps Performed

1. Installed Wireshark.
2. Selected the active network interface.
3. Started packet capture.
4. Generated network traffic by:
   - Browsing websites
   - Using ping commands
5. Stopped the capture after approximately 1 minute.
6. Applied protocol filters.
7. Identified multiple protocols.
8. Exported the capture as a `.pcap` file.

## Protocols Identified

### 1. DNS
- Used for domain name resolution.
- Converts domain names into IP addresses.

### 2. TCP
- Connection-oriented protocol.
- Provides reliable data transfer.

### 3. HTTP/HTTPS
- Used for web communication.
- HTTPS encrypts transmitted data.

## Sample Findings

| Protocol | Purpose |
|-----------|-----------|
| DNS | Domain name resolution |
| TCP | Reliable transport |
| HTTP/HTTPS | Web traffic |

This task helped me understand packet capture, protocol analysis, network communication, and troubleshooting techniques using Wireshark.
